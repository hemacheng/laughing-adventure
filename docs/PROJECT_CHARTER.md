# 專案章程：心境軌跡 (MindTrack)

- **版本**: 1.0
- **作者**: 產品策略顧問

---

## 1. 專案願景 (Project Vision)

打造一個領先的個人成長工具，賦能使用者透過數據洞察，深刻理解個人行為與情緒健康的內在連結，最終將自我覺察轉化為一種毫不費力的日常實踐，實現更平衡、更有意義的生活。

## 2. 主要目標 (Goals)

1.  **市場驗證與早期用戶獲取 (MVP Launch & Adoption)**: 在四個月內成功推出最小可行性產品 (MVP)，並在產品上線後的首兩個月內，獲取 1,000 名活躍註冊使用者。
2.  **核心價值主張驗證 (Value Proposition Validation)**: 在上線後六個月內，達成新使用者第四週留存率 (Week 4 Retention) 30% 以上，證明「關聯性洞察」功能確實能吸引並留住目標客群。
3.  **功能黏著度驗證 (Feature Engagement)**: 確保在活躍使用者中，至少有 50% 的使用者每週至少與「關聯性洞察」圖表互動一次，確認核心功能
的實用性與吸引力。

## 3. 成功衡量指標 (Success Metrics)

| 主要目標 | 關鍵衡量指標 (KPIs) |
| :--- | :--- |
| **市場驗證與早期用戶獲取** | - 每月活躍使用者數 (MAU)<br>- 使用者註冊轉換率 (%) |
| **核心價值主張驗證** | - 新使用者留存率 (Week 1, Week 4)<br>- 淨推薦值 (NPS) |
| **功能黏著度驗證** | - 「關聯性洞察」功能的點擊率 (CTR)<br>- 使用者平均每日/每週 Session 次數 |

## 4. 專案範圍 (Scope)

#### 範圍內 (In-Scope):

- **使用者基礎功能**: 安全的會員註冊與登入系統。
- **核心記錄功能**:
    - 每日心情記錄（可選用量化指標，如 1-5 分，並搭配文字標籤）。
    - 自定義習慣追蹤（可設定每日/每週目標）。
- **核心洞察功能**:
    - 將心情與習慣數據並列於時間軸上的視覺化儀表板。
    - 初版「關聯性洞察」引擎：自動分析單一習慣與心情分數的基礎關聯性，並以圖表呈現。
- **平台**: 優先專注於 Web 應用程式 (桌面與行動裝置優先的響應式設計)。

#### 範圍外 (Out-of-Scope) for MVP:

- 社交或社群功能（如：好友、分享、排行榜）。
- 第三方應用程式整合（如：行事曆、健康手環數據）。
- 原生 App 開發 (iOS/Android)。
- 進階 AI 預測模型或個人化內容推薦（如：文章、冥想引導）。
- 企業版或 B2B 解決方案。

## 5. 核心功能 (Core Features) - MVP

- **使用者身份驗證 (Authentication)**:
    - 提供 Email/密碼註冊與登入。
    - 忘記密碼功能。
- **心情日誌 (Mood Journal)**:
    - 提供一個簡單、直觀的介面，讓使用者能在 30 秒內完成當日心情記錄。
    - 包含 1-5 分的評分系統與可選填的文字備註。
- **習慣追蹤器 (Habit Tracker)**:
    - 允許使用者新增、命名自定義的習慣。
    - 提供每日「完成/未完成」的打卡功能。
- **視覺化儀表板 (Visualization Dashboard)**:
    - 以簡潔的線性圖或長條圖，在同一時間軸上展示心情分數的起伏與習慣完成的狀況。
- **關聯性洞察 (Correlation Insight)**:
    - 系統後端每週自動運行一次分析。
    - 向使用者展示一個最顯著的正向或負向關聯，例如：「我們發現，在你『有運動』的日子裡，你的平均心情分數提升了 15%。」

## 6. 初步假設與風險 (Assumptions & Risks)

#### 關鍵假設 (Assumptions):

- **數據輸入意願**: 假設使用者願意為了獲得洞察，而持續地記錄每日心情與習慣數據。
- **洞察價值性**: 假設系統產生的「習慣-心情」關聯性洞
察，對使用者來說是新鮮、有價值且足夠觸動人心的。
- **簡潔性偏好**: 假設目標客群確實偏好一個專注、簡潔的工具，勝過功能繁雜的綜合性平台。

#### 潛在風險 (Risks):

- **數據稀疏風險 (Data Scarcity Risk)**: 使用者因忘記、懶惰或生活忙碌而未能持續記錄，導致數據不足，無法產生有意義的洞察。
    - *初步緩解策略*: 設計無壓力的提醒機制、簡化輸入流程至極致。
- **洞察平庸風險 (Insight Triviality Risk)**: 產生的洞察可能過於淺顯（例如：「你週末心情比較好」），無法帶來「啊哈時刻 (Aha Moment)」，導致使用者流失。
    - *初步緩解策略*: 在演算法設計初期，優先尋找非直觀的關聯性，並提供篩選功能。
- **市場競爭風險 (Market Risk)**: 個人成長與健康類的應用市場已趨於飽和，我們的獨特價值主張可能不夠突出，難以獲取用戶。
    - *初步緩解策略*: 早期專注於利基市場的行銷，並透過卓越的 UX/UI 設計建立口碑。
- **數據隱私風險 (Privacy Risk)**: 專案處理的是高度敏感的個人情緒與行為數據，任何隱私洩露都將摧毀使用者信任。
    - *初步緩解策略*: 從開發第一天起就導入最高標準的數據加密與安全措施，並提供透明的隱私權政策。
