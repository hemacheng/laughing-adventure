## 核心使用者故事 (Core User Stories)

以下 5 個使用者故事定義了新使用者從註冊到首次體驗產品核心價值的關鍵路徑 (Critical Path)。

---

### 1. 使用者註冊

-   **使用者故事**:
    > 身為一位**追求個人成長的新使用者**，我想要**用 Email 快速建立一個帳號**，以便**我能安全地開始記錄我的個人數據**。

-   **驗收標準 (Acceptance Criteria)**:
    1.  `Given` 我在註冊頁面，`When` 我輸入一組有效的 Email 和安全的密碼並點擊「註冊」，`Then` 我的帳號應該被建立，並且我會自動登入系統。
    2.  `Given` 我在註冊頁面，`When` 我嘗試使用一個已經存在的 Email 註冊，`Then` 我應該會看到一則錯誤訊息，提示我該 Email 已被使用。
    3.  `Given` 我在註冊頁面，`When` 我輸入一個格式不正確的 Email，`Then` 我應該會看到一則行內錯誤訊息，指出 Email 的格式有誤。

### 2. 建立第一個習慣

-   **使用者故事**:
    > 身為一位**首次登入的使用者**，我想要**在完成註冊後被引導建立我的第一個習慣**，以便**我能立即設定目標，開始我的成長旅程**。

-   **驗收標準 (Acceptance Criteria)**:
    1.  `Given` 我是第一次成功註冊並登入，`When` 我到達主儀表板頁面時，`Then` 系統應透過一個彈出視窗或引導提示，邀請我新增我的第一個習慣。
    2.  `Given` 我在新增習慣的引導中，`When` 我輸入習慣名稱「每日冥想」並儲存，`Then` 這個新習慣應該會出現在我的儀表板習慣清單中。
    3.  `Given` 我正在建立一個新習慣，`When` 我沒有輸入任何名稱就嘗試儲存，`Then` 儲存按鈕應該是無效的，並提示我必須輸入名稱。

### 3. 記錄每日心情

-   **使用者故事**:
    > 身為一位**注重自我覺察的使用者**，我想要**透過一個非常簡單的介面記錄我今天的心情**，以便**我能毫不費力地追蹤自己的情緒變化**。

-   **驗收標準 (Acceptance Criteria)**:
    1.  `Given` 我已經登入並位於儀表板，`When` 我點擊「記錄今日心情」按鈕，`Then` 我應該會看到一個包含 1-5 分評分選項的簡潔介面。
    2.  `Given` 我在心情記錄介面，`When` 我選擇了「4 分」並點擊儲存，`Then` 介面應關閉，並顯示一個成功記錄的確認訊息。
    3.  `Given` 我今天已經記錄過心情，`When` 我再次訪問儀表板，`Then` 「記錄今日心情」的按鈕應該會變為「編輯心情」或顯示為已完成狀態。

### 4. 完成一次習慣打卡

-   **使用者故事**:
    > 身為一位**致力於建立慣性的使用者**，我想要**在我完成習慣後，能輕鬆地將其標示為完成**，以便**獲得即時的成就感並記錄我的進度**。

-   **驗收標準 (Acceptance Criteria)**:
    1.  `Given` 我的儀表板上有一個名為「每日冥想」的習慣，`When` 我點擊該習慣旁的核取方塊，`Then` 該方塊應顯示為已勾選狀態，並有明確的視覺變化（如顏色改變或劃掉文字）。
    2.  `Given` 我已經勾選了「每日冥想」的習慣，`When` 我重新整理頁面，`Then` 該習慣應保持在已勾選的狀態。
    3.  `Given` 我不小心勾選了一個習慣，`When` 我再次點擊那個已勾選的核取方塊，`Then` 它應該會恢復到未勾選的狀態。

### 5. 檢視初步數據

-   **使用者故事**:
    > 身為一位**剛開始使用的使用者**，我想要**在儀表板上看到我記錄下的心情和已完成的習慣**，以便**我能直觀地確認我的努力有被系統追蹤**。

-   **驗收標準 (Acceptance Criteria)**:
    1.  `Given` 我今天記錄了心情為「5 分」，並完成了「每日冥想」的習慣，`When` 我查看儀表板的圖表區塊，`Then` 我應該能在今天的日期上看到一個代表 5 分的數據點，以及一個代表習慣完成的標記。
    2.  `Given` 我在過去一週都沒有任何記錄，`When` 我查看儀表板的圖表區塊，`Then` 該區塊應顯示一個空白狀態，並有文字鼓勵我開始記錄。
    3.  `Given` 我已經連續三天記錄了數據，`When` 我查看圖表，`Then` 我應該能看到一條線或長條圖將這三天的心情數據點連接起來。

## 使用者流程圖 (User Flow Diagram)

此流程圖聚焦於「**新使用者從進入網站到完成第一次習慣打卡**」的關鍵路徑。

```mermaid
graph TD
    A[訪問產品首頁] --> B{點擊「免費註冊」};
    B --> C[顯示註冊表單];
    C --> D{填寫表單並提交};
    D --> E{帳號資訊是否有效?};
    E -- 否 --> F[顯示錯誤訊息];
    F --> C;
    E -- 是 --> G[建立帳號並自動登入];
    G --> H[<b style='font-style: italic;'>關鍵時刻: 啟用 (Activation)</b><br/>引導使用者建立第一個習慣];
    H --> I{使用者定義習慣並儲存};
    I --> J[儀表板顯示新建立的習慣];
    J --> K{使用者在線下完成習慣後<br/>回到網站點擊核取方塊};
    K --> L[<b style='color:#2E7D32;'>價值實現!<br/>第一次成功追蹤</b>];
```

## 流程圖詳細說明

以下針對上述流程圖的關鍵步驟，說明其背後的「WHY」，以確保設計決策能對應使用者動機與業務目標。

1.  **步驟 A → G (標準註冊流程)**
    -   **WHY**: 我們的核心是提供個人化的數據洞察，因此一個獨立、安全的帳號是必要的。此流程旨在盡可能降低註冊阻力，同時透過即時驗證確保基本資料的正確性，為後續的個人化體驗打下基礎。

2.  **步驟 H (關鍵時刻：引導建立第一個習慣)**
    -   **WHY**: 這是使用者「啟用 (Activation)」的關鍵一步。與其讓使用者面對一個空白的儀表板不知所措，我們主動引導他們完成最有價值的第一個動作。這不僅能立即教育使用者 App 的核心功能，更能創造「初始投資」，提高使用者次日返回的機率。

3.  **步驟 I → J (定義並展示習慣)**
    -   **WHY**: 讓使用者「自己」命名習慣，能瞬間建立個人連結與擁有感。將這個目標清晰地展示在儀表板上，如同一個待辦事項，為使用者提供了明確的行動召喚 (Call to Action)。

4.  **步驟 K → L (完成第一次打卡)**
    -   **WHY**: 這是使用者首次感受到「微小成就」的時刻。點擊核取方塊這個簡單的動作，提供了即時的正向反饋。這是整個「行為-記錄-洞察」循環中的第一個閉環，也是使用者從「被動瀏覽」轉向「主動參與」的轉捩點。完成這個動作，使用者才算真正「體驗」到了產品的基礎價值。
